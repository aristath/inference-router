Aliases

Stable, canonical names (e.g. planner, coder, fast) that clients request while you swap the underlying model from here. An alias can also point at another alias — e.g. several aliases sharing one default; repointing default moves them all. Deleting a model or alias leaves anything referencing it unassigned, ready to repoint. Aliases always resolve regardless of the exposure switch — the switch only changes what /v1/models advertises.
Unique. Lowercase letters, digits, . _ -. Can't collide with a model id. Renaming is allowed.
