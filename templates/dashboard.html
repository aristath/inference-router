For GGUF, the absolute path to the .gguf file. For Safetensors, the directory that contains the model files.
Which GPU(s) the draft model runs on. Passed verbatim as -devd. For single-target + draft on 3× R9700, isolating the draft to a dedicated GPU (e.g. Vulkan1) avoids VRAM contention with the target's layer-split. Leave blank to let llama.cpp pick.
Enter a model path to get a VRAM estimate and slider. How many tokens (prompt + reply combined) the model can keep in mind for a single conversation. Bigger = remembers more, but uses dramatically more VRAM. Roughly 1 token ≈ ¾ of an English word.
How adventurous the model is when picking the next word. 0 = always picks the single most likely word (deterministic, repeatable). 0.7 = balanced. 1.5+ = wild and creative, sometimes incoherent. Use low values for code and facts, higher for brainstorming and fiction.
"Nucleus sampling." Only consider next-word candidates from the smallest group whose probabilities add up to this fraction. 0.95 is the common default — ignore the bottom 5% of unlikely guesses. 1.0 disables the filter.
Only consider the K most likely next words at each step. 40 is a common default — narrow enough to stay on-topic, wide enough to feel natural. Lower (e.g. 20) = more focused; higher (e.g. 100) = more variety. 0 disables the filter.
Drop any candidate word that's less than this fraction as likely as the top candidate. 0.05 is a common quality floor — kills the obviously-wrong guesses. 0 disables the filter.
Penalize any token that has already appeared at least once, regardless of how many times. llama.cpp default 0.0 (disabled). Positive values discourage re-use of already-seen words; useful to break repetition loops on some Qwen3 variants (commonly 1.5).
Multiplicative penalty on repeated token sequences. llama.cpp default 1.0 (disabled). 1.1 is a mild nudge against loops; values >1.2 start hurting quality.

llama.cpp options

Backend flags for llama-server. Applied only when Weights format is GGUF. Extra arguments below override any of these (llama.cpp takes the last value for a repeated flag).
Faster attention with lower VRAM. Safe to enable on modern builds; a must for longer contexts.
Pin the model in RAM so it can never be swapped out. Use it if your system swaps under pressure and inference stalls.
Load the full model into RAM up front instead of memory-mapping it from disk. Slower to start; avoids page-cache churn on fast NVMe.
How many requests this model can handle concurrently. Each slot gets context / N tokens. 0 = leave at llama.cpp's default (usually 1). Useful >1 for multi-agent workflows.
Number of model layers to offload to the GPU. 0 = leave the flag off (default). The slider's max follows the model's actual layer count once the GGUF metadata loads.
How the model is distributed across GPUs. layer is the safe default — each GPU holds a contiguous range of layers, low inter-GPU traffic. row splits each tensor row-wise. tensor is full tensor parallelism — generally the fastest mode when supported, but bandwidth-sensitive. none forces a single GPU. Note: the Vulkan backend in practice only supports none and layer; row and tensor work with the ROCm build.
Comma-separated fractions per GPU (e.g. 0.5,0.5,0 to use only the first two of three cards). Leave blank and the orchestrator picks the smallest GPU subset whose free VRAM fits the model. Filled in manually pins the split exactly.
Index of the GPU that holds small intermediate tensors (scalars, temp buffers). Usually irrelevant for layer split; matters more for row. Leave blank for llama.cpp's default.
CPU threads llama-server uses for generation. llama.cpp default -1 (auto); set to your physical core count (e.g. 16) for predictable behavior. Leave blank for the default.
Maximum KV cache held on CPU (MiB). llama.cpp default 8192; -1 = no limit, 0 = disable CPU-side caching. Leave blank for the default.
Controls how llama-server parses thought tags in the model's output back into the OpenAI response body. These are the only four values llama.cpp accepts.
Token budget for the thinking phase. llama.cpp default -1 (unrestricted), 0 ends thinking immediately, N>0 caps thinking at N tokens. Leave blank for the default.
JSON object passed into the model's Jinja chat template. Family-specific keys: Qwen3 uses {"enable_thinking":bool}; gpt-oss uses {"reasoning_effort":"low"|"medium"|"high"|"none"}; Qwen3.6 uses {"preserve_thinking":true}. Leave blank to pass nothing.
Quantize the KV cache's keys to shrink VRAM. q8_0 is almost free; q4_0 can hurt quality on long contexts.
Quantize the KV cache's values. Same tradeoffs as K. Requires flash attention to be enabled if not f16.

Speculative decoding

Pick a draft-role model to run alongside this target. The draft's own config (GGUF path, context, cache quant, GPU placement) comes from its model entry; the knobs below tune how hard this target drives it.
Only models with role=Draft appear here. Same-family drafts (e.g. Qwen3.5-2B for Qwen3.5-27B) get high acceptance rates; cross-family drafts don't.
How many tokens the draft generates per speculative step. Higher = bigger potential speedup, but more wasted work on rejection. llama.cpp default 16. Leave blank for default.
Minimum draft length before submitting to the target — short drafts are submitted immediately when low. Leave blank for default.
Probability floor for the draft's greedy sampling. Drafts below this confidence are truncated. Higher = safer drafts, lower speedup.
Number of snapshot slots for rollback on partial draft rejection. Required > 0 for hybrid-recurrent targets (Qwen3.5 dense) so partial-accept rollback uses snapshot/restore instead of seq_rm. Typical 4.
How often to checkpoint during prefill. -1 for default, 0 disables. Leave blank for default.
Raw flags appended to the inference server's command line — use for anything not represented as a field above. One argument per line.
